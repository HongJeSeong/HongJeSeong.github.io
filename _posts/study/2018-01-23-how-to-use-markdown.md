# 마크다운(Markdown) 작성법
개발에서 필요한, 깃허브(github)를 사용하다 보면 마크다운이 필요하다.
깃허브에 자신의 레포지토리에 관한 설명이나, 문서를 작성할때 필요하게 된다.

## 마크다운의 장점
	* 간결함.
	* text로 용량이 적다
	* 변경이력 관리에 수월
	* 지원하는 플랫폼, 프로그램이 다양하다.

## 마크다운의 단점
	* 표준이 없다.
	* 모든 html 마크업을 대신하지 못한다.

## 헤더(Header)
html의	``` <h1>~<h6>``` 헤더와 같다.
```
# This is a H1
## This is a H2
### This is a H3
#### This is a H4
##### This is a H5
###### This is a H6
```
